# Pages 공통 프롬프트 가이드라인

- 문서 버전: 1.0.0
- 작성자: GitHub Copilot (프로젝트 가이드용 자동 생성)
- 마지막 업데이트: 2025-12-04

---

## 1. 목적

이 문서는 `Pages` 폴더 내 기존 및 향후 생성될 페이지(`howtodistribute`, `showmethemoney`, `moneydash` 등)를 위한 공통 프롬프트 규칙과 템플릿을 정의합니다. 이 규칙은 페이지 생성, 수정, 로컬라이제이션, 접근성, 분석 및 배포 관련 일관성을 보장하기 위해 사용됩니다.

## 2. 적용 대상

- `Pages/` 폴더 내의 모든 HTML, CSS, JS, 문서 파일
- 새로 추가될 페이지와 해당 리소스

## 3. 목표

- 일관된 페이지 메타데이터 구조 유지
- 개발자 및 자동화(예: AI 보조 생성)가 따를 수 있는 명확한 프롬프트 템플릿 제공
- 접근성(A11y), 성능, 개인정보 보호 준수
- 다국어(로컬라이제이션) 지원 규칙 포함

## 4. 공통 메타데이터(파일 상단에 주석 또는 프론트매터 권장)

각 페이지는 다음 메타데이터 중 최소 부분을 포함해야 합니다(HTML 주석 또는 별도 JSON/YAML):

- title: 페이지 제목 (locale별로 번역)
- description: 간단한 설명(SEO, locale별)
- route: 페이지 상대 경로(예: `/showmethemoney`)
- layout: 사용할 레이아웃(옵션)
- scripts: 필요한 스크립트 파일 목록
- styles: 필요한 스타일시트 파일 목록
- analytics: 사용되는 분석/태그 매니저 ID (없으면 `none`)
- locale: 기본 로케일 (예: `ko`)
- author: 작성자
- lastUpdated: YYYY-MM-DD

예시 (HTML 주석):
<!--
  title: 복권 번호 생성기
  description: 로또 및 연금복권 번호를 생성하는 간단한 도구
  route: /showmethemoney
  layout: default
  scripts: ["showmethemoney.js"]
  styles: ["showmethemoney.css"]
  analytics: GTM-KLNBMN45
  locale: ko
  author: NyanPunch
  lastUpdated: 2025-12-04
-->

## 5. 공통 프롬프트 템플릿

새 페이지를 만들거나 기존 페이지를 업데이트할 때 사용할 기본 프롬프트 구조:

1) 페이지 생성 AI 프롬프트 (구성/초안 생성용)
- 목적: 이 페이지의 목표(한 줄)
- 대상 사용자: 누구를 위한 페이지인지
- 핵심 기능: 필요한 인터랙션/입력/출력
- 메타데이터: title, description, route, locale, analytics
- 접근성 요구사항: 키보드 네비게이션, 레이블, ARIA 등
- 성능/경량화: 외부 리소스 최소화, 이미지 lazy-load 등
- 보안/프라이버시: 개인 데이터 수집 금지 또는 명시
- 로컬라이제이션: 번역이 필요한 텍스트 목록
- 예시 입력/출력: 사용자가 기대하는 동작 예시

2) UI/JS 구현 프롬프트 (구현 세부)
- 필요한 DOM 요소 및 id/클래스
- 이벤트 핸들러 및 동작(예: 버튼 클릭 -> 번호 생성)
- 입력 유효성 검사 규칙
- 에러 메시지와 로컬라이즈드 텍스트
- 테스트 시나리오(간단한 사용 흐름)

3) 문서/메타 정보 프롬프트
- README 단락(한 문단)
- 포함 스크린샷 또는 예시 코드 블록(옵션)
- 배포 힌트(analytics 포함 여부, ads, ad networks)

## 6. 접근성(A11y) 규칙

- 모든 인터랙티브 요소에 레이블 제공(label 또는 aria-label)
- 키보드만으로 조작 가능해야 함(포커스 순서 명확)
- 색 대비는 WCAG AA 이상 권장
- 시맨틱 HTML 사용(h1~h6, main, nav, footer 등)

## 7. 성능 및 리소스 관리

- 외부 스크립트는 async 또는 defer 사용 권장
- CSS는 가능한 한 병합/축소 권장
- 광고(예: Adsense)를 포함할 경우 페이지 로딩 영향을 고려해 비동기 로드

## 8. 프라이버시 및 광고

- 페이지에 개인/민감 데이터 수집 로직을 두지 마세요
- 광고 네트워크를 사용할 경우 개인정보·쿠키 정책 표기 필요
- GTM/Analytics ID는 메타데이터 필드에 명시

## 9. 로컬라이제이션

- 모든 사용자 메시지, 라벨, 설명은 로케일별로 별도 리소스에 분리
- 기본 언어는 `ko`로 작성하고, 필요한 경우 영어(`en`) 리소스 추가

## 10. 코드 스타일 및 커밋 메시지

- HTML/CSS/JS는 프로젝트 기존 스타일을 준수
- 커밋 메시지 예시: `pages(showmethemoney): add lotto generator and metadata`
- 변경 시 `lastUpdated` 메타데이터 갱신

## 11. 예시: `showmethemoney`용 간단 프롬프트

- 목적: 로또 6/45 및 연금복권 720+ 번호를 생성하는 프론트엔드 도구
- 대상 사용자: 일반 사용자(모바일/데스크탑)
- 핵심 기능: 사용자 입력(포함할 번호 등), 번호 생성, 결과 표시, 입력 유효성 검사
- 접근성: 버튼 및 입력에 레이블, 결과는 스크린리더가 인식 가능하도록 텍스트로 제공
- 메타데이터: see #4 예시

## 12. 향후 페이지 확장 팁

- 페이지 공통 컴포넌트(헤더, 푸터, analytics 스니펫)는 재사용 가능하게 구성
- 페이지별 JS는 모듈화하여 충돌을 최소화
